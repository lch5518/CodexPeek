# 작업 표시줄 글라스 위젯 디자인

## 목적

Codex 사용량 조회, RPC, 폴링, 설정 및 도메인 모델을 변경하지 않고 Windows 작업 표시줄 위젯만 더 작고 명확한 주간 사용량 표시기로 개선한다. 처음 보는 사용자가 2초 안에 `주간 사용량`과 사용한 비율, 위험 상태를 이해할 수 있어야 한다.

## 범위

- 작업 표시줄 위젯의 크기, 배치, 렌더링, 상태 표현 및 툴팁을 변경한다.
- 부동창, 트레이 메뉴, 사용량 조회와 인증 흐름은 유지한다.
- 기존 `WidgetViewModel`을 소비하되 작업 표시줄에서는 `secondary`를 주간 사용량으로 우선 선택한다. `secondary`가 없는 예외적인 응답에서는 존재하는 사용량 창을 안전하게 표시한다.
- `usedPercent`는 사용한 비율이다. 남은 비율은 UI에서 `max(100 - usedPercent, 0)`으로만 계산한다.

## 시각 설계

- 기준 크기는 208×48 논리 픽셀이고 가로 폭은 콘텐츠에 따라 변하지 않는다.
- 한 행만 사용한다.
  - 위쪽: 6px 상태 점, `주간 사용량`, 오른쪽 고정 퍼센트
  - 아래쪽: 좌우 여백 11px을 둔 3px 진행 막대
- 글꼴은 `Segoe UI Variable`을 우선하고 `Segoe UI`로 대체한다.
- 본문은 11~12px, 퍼센트는 medium 또는 semibold로 표시한다.
- 배경은 약 68%의 어두운 반투명 소재를 사용하고 10px 모서리 반경을 적용한다.
- 현재 Explorer 자식 창에는 공식 DWM blur API를 적용할 수 없으므로 비공식 합성 API를 사용하지 않는다. 기존 `UpdateLayeredWindow` 경로에서 반투명 배경, 둥근 알파 마스크, 미세 노이즈와 부드러운 가장자리 하이라이트로 frosted fallback을 구현한다.
- 단단한 테두리와 검은 그림자는 사용하지 않는다. 가장자리에는 약 5% 흰색 하이라이트만 둔다.

## 상태 표현

작업 표시줄 전용 표현 임계값을 사용하며 도메인 `UsageLevel`은 변경하지 않는다.

- 0~69%: 차분한 녹색
- 70~89%: 주황색
- 90% 이상: 빨간색
- 로딩: 중립 회색
- 조회 실패: 작은 경고 기호와 중립/경고색

색상 외에도 툴팁에 상태 설명을 포함한다.

## 상호작용

- 마우스 진입 시 150ms 동안 배경 밝기만 소폭 증가한다. 크기, 좌표, 텍스트 위치는 움직이지 않는다.
- 마우스 이탈 시 같은 경로로 원래 밝기로 돌아간다.
- 위젯 자체에 새로운 클릭 동작을 추가하지 않으므로 포인터 커서를 사용하지 않는다.
- 네이티브 툴팁은 기준 기간, 사용 비율, 남은 비율, 초기화 정보를 표시한다. 긴 정보는 기본 위젯에 넣지 않는다.

## 구현 구조

- `src/windows/taskbar.rs`: 작업 표시줄 크기를 208 논리 픽셀 폭으로 변경한다.
- `src/windows/native/platform.rs`: 작업 표시줄 전용 렌더러를 부동창 렌더러와 분리하고 글라스 합성, 둥근 알파 마스크, 상태색, hover 및 툴팁을 담당한다.
- `src/windows/widget.rs`: 필요한 경우 작업 표시줄용 순수 레이아웃 계산을 추가한다.
- `src/windows/mod.rs` 또는 현지화 모듈: UI 문구만 추가하며 사용량 데이터 구조는 유지한다.

## 오류와 fallback

- 향후 Windows App SDK 또는 최상위 오버레이 구조를 도입하기 전까지 네이티브 Acrylic은 비목표로 둔다.
- 반투명 레이어드 렌더링도 실패하면 기존 부동창 fallback을 유지한다.
- 툴팁 생성 실패는 기본 위젯 표시를 막지 않는다.
- 주간 데이터가 없으면 로딩 또는 오류 상태를 표시하되 창 폭은 유지한다.

## 검증

- 96, 120, 144, 192 DPI에서 너비, 여백, 상태 점, 퍼센트 고정 영역과 진행 막대가 경계 안에 있는지 순수 레이아웃 테스트를 추가한다.
- 0, 69, 70, 89, 90, 100 초과 사용률의 작업 표시줄 전용 색상 상태를 테스트한다.
- 주간 우선 선택과 단일 창 fallback을 테스트한다.
- 로딩, 정상, 경고, 위험, 오류 렌더링을 확인한다.
- 전체 테스트, Clippy, 릴리스 빌드 후 실제 Windows 11 작업 표시줄 캡처로 208×48 렌더링과 투명 가장자리를 확인한다.

## 비목표

- 사용량 API, 인증, 폴링, 도메인 임계값, 부동창 및 트레이 UI를 재설계하지 않는다.
- 사용자 정의 테마, 복잡한 애니메이션, 상세 패널이나 새로운 설정 화면은 추가하지 않는다.
